# Backlog Inicial Priorizado

A continuación se detallan las Historias de Usuario (HU) priorizadas para el desarrollo del Sistema de Gestión de Incidencias en un plazo estimado de 3 meses (Producto Mínimo Viable).

| ID        | Prioridad | Historia de Usuario                                                                                                                                    | Criterios de Aceptación                                                                                                                  |
| :-------- | :-------: | :----------------------------------------------------------------------------------------------------------------------------------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------- |
| **HU-01** |   Alta    | **Como** Administrador,<br>**quiero** poder dar de alta usuarios y asignarles un rol,<br>**para** que puedan acceder al sistema de forma segura.       | - El sistema exige email y contraseña.<br>- La contraseña se guarda encriptada.<br>- Los roles disponibles son: Usuario, Técnico, Admin. |
| **HU-02** |   Alta    | **Como** Usuario,<br>**quiero** hacer login con mis credenciales,<br>**para** acceder a mi panel personalizado.                                        | - Si falla la contraseña, muestra error genérico.<br>- Al entrar, redirige al panel según el rol.                                        |
| **HU-03** |   Alta    | **Como** Usuario,<br>**quiero** crear un ticket de incidencia indicando categoría y prioridad,<br>**para** reportar un problema al departamento de TI. | - Título y descripción son obligatorios.<br>- El ticket nace siempre en estado "Abierto".                                                |
| **HU-04** |   Alta    | **Como** Técnico,<br>**quiero** ver un listado de todos los tickets abiertos,<br>**para** saber qué trabajo hay pendiente.                             | - El listado muestra Título, Creador y Fecha.<br>- Se puede ordenar por Prioridad.                                                       |
| **HU-05** |   Media   | **Como** Técnico,<br>**quiero** cambiar el estado de un ticket a "En Progreso" o "Resuelto",<br>**para** mantener informado al usuario del avance.     | - Solo un técnico asignado puede cambiar el estado.<br>- Si pasa a "Resuelto", exige un comentario final.                                |
| **HU-06** |   Media   | **Como** Usuario,<br>**quiero** poder adjuntar una captura de pantalla al crear el ticket,<br>**para** que el técnico entienda mejor el fallo.         | - Solo permite formatos de imagen (.png, .jpg).<br>- El tamaño máximo es de 5MB.                                                         |
| **HU-07** |   Baja    | **Como** Administrador,<br>**quiero** exportar el listado de tickets resueltos a CSV,<br>**para** hacer informes mensuales de rendimiento.             | - El archivo CSV se descarga con un clic.<br>- Incluye tiempo de resolución.                                                             |
